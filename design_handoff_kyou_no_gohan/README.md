# Handoff: きょうのごはん（レシピアプリ）

> **読む人へ** — このフォルダはデザイナー（Claude）が作成した HTML プロトタイプ + 実装ガイドです。VSCode + Claude Code で実装する想定で書かれています。

---

## 0. はじめに / Overview

「きょうのごはん」は、冷蔵庫の中身を**写真・画像・テキスト・音声**から入力すると、AI が食材を検出して **3 つのレシピを提案**するスマホアプリです。

- **ターゲット**: 節約志向の主婦・主夫
- **トーン**: 温かみのある手作り感（ナチュラル、暖色系、紙ノート風）
- **プラットフォーム**: iOS（iPhone 15 系のサイズで設計）

---

## 1. これらのファイルについて / About the design files

**`design/` 配下の HTML/JSX ファイルは「実装の参照用デザイン」です。** 本番コードとしてコピーするためのものではありません。

- 表現に React + Babel スタンドアロン + 自前のスタイルトークンを使っていますが、これは「Figma の代わりにブラウザで見られるモック」程度の位置づけです。
- ターゲット環境（SwiftUI / React Native / Flutter / 既存の React など）の**慣習・コンポーネントライブラリに合わせて再実装**してください。
- 既存環境が無い場合は、最も適切なフレームワークを選んで実装してください。

**表示方法**: `design/きょうのごはん.html` をブラウザで開くと、Figma 風のキャンバスに全 11 画面が並びます（パン: ドラッグ、ズーム: トラックパッドピンチ / Cmd+ホイール、フォーカス: アートボードのラベルクリック）。

### Typography

```
見出し（Heading）:    Zen Old Mincho      Google Fonts
本文（Body）:         Zen Kaku Gothic New  Google Fonts
等幅（Mono / 数字）:  JetBrains Mono       Google Fonts
```

サイズ階段（px）:
- 数字大: 38 / 34 / 30 / 28 / 26 / 24 / 22
- 見出し: 18 / 16 / 15
- 本文: 14 / 13
- 補助: 12 / 11 / 10 / 9

文字間:
- 日本語見出し: `letter-spacing: 0.01〜0.04em`
- 英字ラベル（CAPS）: `letter-spacing: 0.12〜0.22em`

`line-height`:
- 見出し: 1.3〜1.4
- 本文: 1.6〜1.75

### Spacing / Radius

- 画面パディング水平: **22px**（強い縦リズム）
- カード radius: **14〜22px**
- ボタン radius: 完全な pill（`9999px`）か `26px`
- セクション間: 16〜28px

### Texture

紙のノイズ感を出すため、`tokens.jsx` の `PAPER_NOISE`（SVG `feTurbulence` を data-URL 化）を `background-image` として、`background-blend-mode: multiply` で重ねています。実装で省略可能ですが、入れると一気に「手作り感」が出ます。

## 5. インタラクション・アニメーション / Interactions

### グローバル
- すべての押下対象に対し iOS 標準の `active state`（軽い不透明度 + スケール 0.97 程度）
- 画面遷移: iOS の push（右からスライドイン）
- ボトムシート: 下からスライド + 背景にディム + バックドロップブラー

### 4.4 AI 解析中の食材ドロップイン（**目玉インタラクション**）
ユーザーの希望: 「検出した食材を順番にアニメーションで表示」

実装イメージ:
1. シャッターを切ると、暗い画像の上に**約 400〜600ms 間隔**で食材アバターが 1 つずつ現れる
2. 各食材は以下のシーケンス:
   - フェードイン + スケール `0.6 → 1.05 → 1.0`（spring）
   - 同時にラベルピル（食材名 + %）が右から `translateX` スライドイン
   - 検出位置に薄い「パルスリング」（無限ループ、低不透明度）
3. 下部シートのリストにも同期して 1 行ずつ追加される
4. 全食材検出後、自動で「食材一覧・編集」へ遷移（または「次へ」ボタンを出す）

推奨ライブラリ:
- React Native: `react-native-reanimated`
- SwiftUI: `withAnimation(.spring())` + `.transition`
- Web: Framer Motion か CSS `@keyframes`

### 4.3 / 4.4 カメラの「読み込み中」ドット
3 つのドットが順次フェード（`@keyframes kgPulse` 参照、1.2s ループ、0/0.2/0.4s ディレイ）

### 4.7 詳細のプルアップシート
- ヒーロー画像が背景に固定、シートを上にドラッグできる
- スクロール中にヒーローを軽くパララックス（速度 0.5）させると質感が出る

## 6. ステート / State

最低限必要なステート:

```ts
// グローバル
session: { userName: string, fridgeScans: FridgeScan[], savedRecipes: Recipe[] }

// ホーム
- 直近のスキャン履歴（最大3件）

// カメラ → 解析フロー（ウィザード）
captureFlow: {
  step: 'camera' | 'analyzing' | 'review' | 'suggestions',
  capturedImage: ImageRef | null,
  detectedIngredients: Ingredient[],       // { id, name, quantity, confidence, category }
  detectionProgress: number,               // 0..1 で 4.4 のプログレス
  userAddedIngredients: Ingredient[],      // ユーザーが手動で追加・編集したもの
  suggestedRecipes: Recipe[],              // AI 提案 3 件
}

// 詳細
recipeDetail: { recipe: Recipe, missingIngredients: Ingredient[], saved: boolean }

// 設定（未デザインだが必要になる）
userPrefs: { allergies: string[], dislikes: string[], maxCookTime: number, servings: number }
```

データ取得:
- AI 解析: 画像を multipart で API に送信 → 食材配列を受信
- レシピ提案: 食材配列 + ユーザー設定を送信 → レシピ 3 件を受信
- **解析と提案は分割エンドポイントにする**ことを推奨（4.4 のアニメ中にバックグラウンドで提案を fetch しておけば、4.6 への遷移が速い）

## 7. アセット / Assets

### このプロトタイプに含まれる**プレースホルダー**

| 種類 | 現在の表現 | 本実装で必要なもの |
|---|---|---|
| 料理写真（ヒーロー・カード） | ストライプパターン + 「料理写真」モノラベル | **実写の料理写真**（縦長 16:9〜4:3 推奨） |
| 撮影中ビューファインダー | グラデ + ぼかし矩形 | 実カメラフィード（`<Camera>` 等） |
| 食材イラスト | SVG の単純な形（`Veggie` コンポーネント） | そのまま使ってもよいが、**統一感のある食材イラストセット**を発注すると品質が上がる |
| アバター（ホーム挨拶） | 漢字 1 文字の丸 | ユーザー設定画像（フォールバックとして頭文字を残す） |

### フォント
3 つすべて Google Fonts（無料）:
- Zen Old Mincho (400, 500, 600, 700)
- Zen Kaku Gothic New (400, 500, 600, 700)
- JetBrains Mono (400, 500)

## 8. コピー / Copywriting tone

「やさしく、ひらがな多めで、押し付けがましくない」が原則。

- 「今日のごはん、なに作ろう？」○
- 「あなたにぴったりのレシピを提案します」×（よそよそしい）
- 漢字を避ける場面: ボタンラベル・ガイダンス（「冷蔵庫の中身を見せてくれたら、今夜のレシピを 3 つ考えます」）
- 漢字を残す場面: レシピタイトル・食材名（伝統的で美味しそうに見える）
- 数値・単位は半角数字 + 全角単位（`15 分`、`¥280`、`200g`）

## 10. 実装の進め方（おすすめ）

1. **ユーザーにホーム案 A/B/C のどれにするか確認**
2. デザイントークンを移植（§3）
3. 共通アトムを実装（Button、Tag、IconButton、SectionHeader、IngredientChip、StatRow、etc.）
4. オンボーディング → ホーム → カメラ → 解析中 → 食材一覧 → 提案 → 詳細 の縦串フローを動かす
5. テキスト入力・音声入力のフォールバック
6. 保存・お気に入り
7. 設定（アレルギー・苦手食材・人数）
8. 詳細のアニメーション・パララックス
9. **AI 解析中の食材ドロップインアニメーションを丁寧に作り込む**（このアプリの「気持ちよさ」の中心）
