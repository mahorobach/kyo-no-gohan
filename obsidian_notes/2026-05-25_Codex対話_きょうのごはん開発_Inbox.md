### レシピ品質・データベース方針

- アプリの肝は「簡単に美味しい料理」。
- AIにゼロから創作させるだけだと、美味しさの保証が弱い。
- 方針としては、裏側に「定番料理の型」データベースを持ち、AIはその型を選んで調整する役割にするのが良さそう。
- レシピサイトを無断スクレイピングして裏DB化するのは避ける。
  - 利用規約、著作権、データベース権、信頼性の問題がある。
  - 「裏で集めて、表ではAI創作として出す」は避けたい。
- 安全な方向性:
  - 自前の料理パターンDBを育てる。
  - 利用許可のあるオープンデータを使う。
  - 公式APIがあるサービスを使う。
  - レシピサイトは人間レビュー時の参考にし、文章・分量・手順をコピーしない。

#### オレンジページについて

- 品質面ではかなり良い候補。
  - プロ料理家・編集部系のレシピで、「簡単においしい」に合う。
- ただし、公式レシピAPIは見つからなかった。
- 掲載コンテンツの権利は原則オレンジページまたは外部制作者に帰属。
- 勝手にスクレイピングして裏DB化する用途には向かない。
- 現実的な使い方:
  - 正式に問い合わせてライセンス相談。
  - 参考レシピとしてリンク先にする。
  - 人間レビュー時の参考にするが、文章・分量・手順はコピーしない。
- 参考:
  - `https://www.orangepage.net/page/rule`
  - `https://www.orangepage.net/sitepolicy`
  - `https://www.orangepage.net/contact`

#### 料理パターンDB案

- 最初は100件を目標に、後から300件以上へ増やせる構造にする。
- 1件ごとに持つとよさそうな項目:
  - `id`
  - `name`
  - `requiredIngredients`
  - `optionalIngredients`
  - `missingAllowed`
  - `method`
  - `flavor`
  - `categories`
  - `time`
  - `difficulty`
  - `tips`
- 例:

```js
{
  id: 'pork-cabbage-miso-stir-fry',
  name: '豚キャベツ味噌炒め',
  requiredIngredients: ['豚肉', 'キャベツ'],
  optionalIngredients: ['玉ねぎ', 'ピーマン', 'にんじん'],
  method: '炒める',
  flavor: '味噌だれ',
  categories: ['節約', 'がっつり', 'お弁当'],
  time: 15,
  difficulty: 'かんたん',
  tips: ['キャベツは最後に入れる', 'たれは先に混ぜる']
}
```
