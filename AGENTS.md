# AGENTS.md — AI Token Manager

## プロジェクト概要
各AIサービスのトークンリセット時刻を管理し、復帰時刻になったらブラウザ通知を受け取るWebアプリ。

## 技術スタック
- React + Vite
- JavaScript（TypeScriptは使わない）
- CSSモジュール or Tailwind CSS
- データ保存: localStorage
- 通知: Web Notifications API
- Pythonは使用しない

## 開発方針
- 段階的に機能を追加する（一度に全部作らない）
- 各フェーズが動いてから次に進む
- シンプルさを優先する。複雑な設計は避ける
- コメントは日本語で書く

## フェーズ構成

### フェーズ1: 基本UI
- AIサービスの一覧表示
- サービスの追加・削除
- 無料／有料の区別
- localStorageへの保存・読み込み

### フェーズ2: タイマー・時刻管理
- 復帰時刻を「何時何分」で指定
- カウントダウン表示（残り時間）
- 更新日（月次サイクル）の管理

### フェーズ3: ブラウザ通知
- 通知許可のリクエスト
- 復帰時刻になったら通知
- タブを開いている間のみ動作（制約として明示する）

### フェーズ4: AIサイトへのリンク
- サービスのアイコンをクリックするとそのAIサイトを開く
- アイコンは画像URLまたはfavicon取得で表示

## 既知の制約
- ブラウザを閉じると通知は届かない（PWA化は将来対応）
- AIサービス側の残量は自動取得できない（手動入力のみ）
- リセット時刻はユーザーが手動で設定する

## 初期登録サービス
| サービス名 | プラン | リセット種別 | URL |
|---|---|---|---|
| Codex | 有料 | 週次（土曜07:00） | https://Codex.ai |
| Gemini | 無料 | 日次 | https://gemini.google.com |
| Codex | 無料 | 日次 | https://chatgpt.com |

## コーディングルール
- コンポーネントは機能単位で分割する
- ファイル名はPascalCase（例: ServiceCard.jsx）
- 関数名・変数名はcamelCase
- マジックナンバーは定数として定義する
- エラー処理を省略しない

## 指示の受け方
ユーザーは日本語で指示を出す。
実装前に要件を1〜2行で復唱し、確認を取ってから実装すること。
不明点は作業を止めて質問すること。
